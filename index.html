Welcome to the MAGenTA project website

To address the need for (1) an analysis method based on exact fitness contribution of a genomic region and (2) a holistic pipeline that incorporates this novel method with the essentiality test, we developed MAGenTA (Microbial Assessment by Genome-wide Tn-seq Analysis), a bioinformatics pipeline for Tn-seq data analysis. In addition to developing novel methods that employ the fitness assessment, we automated comparative analyses that were previously performed manually and added the essentiality test resulting in a streamlined, uniform pipeline.

In detail, MAGenTA enables analyses to:

(a) calculate fitness across a sliding window, enabling identification of protein domains and non-coding regions (b) assess gene essentiality in an organism based on transposon insertion representation (c) perform bacterial strain comparison to assess how the same gene can have different effects in a strain-dependent manner (d) perform comparative analyses for gene and regional differences across experiments (e) visualize multi-feature tracks for a given genomic region

Descriptions of MAGenTA Tools

SlidingWindow.pl

Description: scans the genome with a fixed size sliding window, calculating a) organismal fitness and b) essentiality based on insertion representation calculations for the genomic region included in the window
Usage: Input .csv file(s) (outputted from calc_fitness.pl) with reference genome. Outputs following files into a single directory:

  • unmatched.txt Genome indexes of insertion sites from Tn-Seq data that could not be matched with a TA site in the given genome at that exact index
  • allTAsites.txt All TA sites that occur in the genome and whether an insertion occurred there
  • nullDist.txt Statistics (size, mean, variance, etc) on null distribution libraries used in the essentials analysis
  • essentialWindows.csv All information about each window (including fitness and p-value for essentiality)
  • essentialWindows.wig Only p-values for each window in wig file for Integrated Genome Viewer (IGV)
  • fitWindows.csv Only fitness-related information for each window
  • fitWindows.wig Only fitness for each window in wig file for Integrated Genome Viewer (IGV)
  • groupedWindows.txt Grouped consecutive windows with same fitness value
  • log.txt If the --log option was selected, all standard output goes here

merge.pl

Description: Merges and sorts multiple comma-separated (.csv) files
Usage: Input .csv files, outputs merge sorted file

countTA.pl

Description: Counts number of TA sites in the genome
Usage: input genome, outputs number of TA sites

customWindow.pl

Description: Given a tab delimited text file of genome region name, start position, and end position, customWindow.pl outputs Tn-Seq stats for those regions: # of mutants, # of unique insertions, # of TA sites, ratio of insertions to TA sites, aggregate fitness value, p-value for essentiality
Usage: input GBK file, fasta file for genome sequence, results files, custom region file

getCoordsGBK.py

Description: Get gene names and coordinates from a GBK file.
Output file can be inputted into Gviz as an annotation track for visualization.
Usage: python getCoordsGBK.py

singleFit.pl

Description: Gets a single fitness value for each insertion based on multiple mutants (lanes)
Usage: perl ../singleFit.pl
